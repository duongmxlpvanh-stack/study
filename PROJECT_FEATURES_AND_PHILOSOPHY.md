# 📋 study管理中心 — 功能清单与设计哲学

> 本文档是项目的**需求规格**，描述"做什么"而非"怎么做"。
> 用于指导重建项目时的功能设计和交互决策。

---

## 一、项目定位

一个面向大学生的**个人学习管理工具**。帮助学生追踪学习进度、管理薄弱知识点、倒计时考试、生成练习试题、用 AI 规划复习日程。

**一句话描述**：打开一个 Dashboard 就能看清全局，敲一条命令就能完成记录。

**目标用户**：在校大学生（一人使用，非团队协作）

---

## 二、用户功能清单

### 2.1 学习进度记录

- 用户用一句话记录"某门课今天学了什么"，自动带上时间戳
- 所有记录按时间倒序排列，可浏览历史
- 可按科目筛选查看

### 2.2 薄弱知识点管理

- 用户记录自己"没搞懂的知识点"
- 每条薄弱点附一个**紧急程度标签**：`紧急`（马上要考） / `不急`（可以以后看） / `考前看`（考前临时抱佛脚）
- 按分类汇总数量，一眼看清有多少"紧急"需要优先处理

### 2.3 考试倒计时

- 用户添加考试名称和日期（如"期末考试 2026-07-15"、"六级 2026-12-15"）
- 系统自动计算距离考试的剩余天数
- 所有考试以倒计时列表展示，按紧急程度着色（红色=临近、黄色=适中、绿色=充裕）
- 支持随时增删考试

### 2.4 科目与资料管理

- 用户登记本学期所学课程（如"高等数学""大学物理"）
- 每门科目自动创建一个独立的资料文件夹
- 用户可将试题、PPT、笔记等文件放入对应文件夹
- 支持一键打开科目文件夹查看资料

### 2.5 学习日记

- 用户每天写学习日记，由外部编辑器打开（VS Code / 记事本等）
- 日记自动保存，含字数统计
- 支持查看指定日期日记、列出最近 N 天日记目录
- 支持**全文搜索**历史日记（通过关键词查找过往记录）
- 支持删除日记

### 2.6 全局仪表板（Dashboard）

- 一个页面汇总所有关键信息：
  - 考试倒计时（哪些考试、还有几天）
  - 课程列表（每门课文件夹中有多少资料）
  - 薄弱点统计（紧急/不急/考前看 各有多少条）
  - 学习统计（累计学习天数、总记录数、连续学习天数、日均记录数）
  - 最近学习进度摘要
  - 最近日记摘要
- 用户打开 Dashboard 就能对整体学习状态一目了然

### 2.7 学习热力图

- 以 GitHub 贡献图风格的**日历热力图**展示过去约 140 天的学习频率
- 每天的颜色深浅代表当天的学习记录数量
- 可按科目筛选（只看某门课的热力分布）
- 悬停/点击可查看具体日期和记录数，点击可直接跳到当天的日记

### 2.8 连续学习统计（Streak）

- 统计**连续学习天数**（当前连续多少天有学习记录）
- 统计累计学习天数和总记录数
- 计算日均学习记录数

### 2.9 行政事务备忘

- 用户记录非学习类的行政事务（如学分认定、考试报名等）
- 可列出查看、支持备忘检索

### 2.10 PDF 试题与讲义生成（可选功能，需额外配置）

- 用户指明科目、章节范围、题数，系统自动调用 AI 出题
- 两种模式：
  - **试题模式（exam）**：生成题目卷 + 答案卷两份 PDF，题号一一对应
  - **讲义模式（study）**：生成知识点复习讲义 PDF，带目录和超链接
- 支持指定难度（基础 / 期末 / 竞赛 / 混合梯度）
- 支持指定题型配比（选择题、填空题、计算题等数量分配）
- 部分科目（高数、物理、概率论）支持自动生成数学配图（函数图像、受力分析等）
- 科目支持**模糊输入**（用户输入"高数"自动识别为高等数学，"大物"→大学物理）
- 可查看某科的章节清单
- 生成后可选自动编译为 PDF
- 可配置默认参数（模式、难度、题数、PDF 引擎等）

### 2.11 Google Drive 云端上传（可选功能，需额外配置）

- 一次浏览器授权后，可将生成的 PDF 上传到 Google Drive
- 文件按 `study_pdfs/<科目>/` 归类存放
- 支持开启"自动上传"（每次生成 PDF 后自动上传）
- 也支持手动上传任意文件并指定所属科目
- 可查看认证状态

### 2.12 AI 学习规划（可选功能，需额外配置）

- AI 综合分析：学习进度、薄弱知识点、即将到来的考试、现有日历忙闲时段
- 自动生成学习计划（哪天什么时间学什么、优先复习什么）
- 生成的计划自动写入 Google Calendar（创建日历事件，含科目、重点内容、优先级）
- 支持配置偏好时间段（如只在 08:00-22:00 安排学习）和默认学习时长
- 支持冲突检测（避开已有日程）
- 可查看近期 AI 生成的学习事件
- 可一键清除 AI 生成的学习事件

### 2.13 交互式命令行（REPL）

- 用户可直接输入命令进入交互模式，无需每条命令前加程序名
- 支持引号包裹含空格的参数
- 常用命令有短别名（如 `dj` = diary）
- 内置帮助系统，用户可以随时查看所有命令

### 2.14 桌面客户端（Windows GUI，可选）

- 提供图形界面替代命令行操作
- 侧边导航栏，所有功能一栏直达
- 仪表板页面展示：统计卡片 + 学习热力图 + 考试倒计时 + 近期日记
- 所有列表页面支持：加载状态、空状态引导、骨架屏
- 支持暗色模式和亮色模式
- 删除操作有确认对话框
- 操作结果有 Toast 通知
- 支持键盘快捷键导航（Ctrl+1~9 切换页面）
- 统计卡片数值有动画效果
- 页面切换有过渡动画

## 三、设计哲学

### 3.1 核心理念

> **好工具的标准：用起来感觉不到它的存在，但少了它立刻不习惯。**
>
> **把精力留给学习本身，管理交给工具。**

### 3.2 极简原则

- **单一入口**：Dashboard 是所有信息的汇聚点，用户日常只需要看它
- **命令即操作**：学习记录就是一句话，不做表单、不填字段
- **不折腾**：学习已经够累了，工具不应该增加认知负担
- **默认够用**：开箱即用的默认值覆盖 90% 场景，高级选项藏在后面

### 3.3 零门槛上手

- **首次运行自动引导**：检测到无配置 → 启动交互式向导，对话式完成设置
- **不要求用户懂技术**：不需要安装运行时、不需要配环境变量、不需要懂命令行
- **出错有解**：错误提示用中文、说人话、给出修复建议（不出堆栈跟踪）
- **双击就能跑**：Windows 用户双击 exe 即可启动

### 3.4 单文件交付

- 整个核心工具就是一个可执行文件，复制到任意电脑即可使用
- 不需要安装器、不需要运行时、不需要依赖
- 可选功能（PDF 生成、Google 服务）才需要额外配置

### 3.5 安全第一

- **凭据绝不硬编码**：API Key、OAuth Token 等敏感信息不在源码中出现
- **凭据不写配置文件**：配置文件可安全分享、纳入 Git，不含任何密钥
- **平台级加密**：Windows 凭据管理器（DPAPI 加密）存储敏感数据
- **OAuth 用 PKCE**：Google 授权使用 PKCE 流程 + 随机 state 防 CSRF

### 3.6 数据可移植

- 学习数据全部以 **Markdown 纯文本文件**存储（人类可读、Git 可版本控制）
- 日记使用 SQLite（单文件数据库，方便备份）
- 整个数据目录复制到新电脑即可继续使用
- 不锁定用户数据（用户随时可以用任何编辑器查看和修改）

### 3.7 功能模块化

- **核心功能**（进度记录、薄弱点、考试、科目、日记）永远可用，零依赖
- **可选功能**（PDF 生成、Google Drive、Google Calendar、AI 规划）按需启用
- 用户不需要某个功能，它就完全不打扰用户

### 3.8 CLI 优先、GUI 可选

- 核心交互是命令行（对键盘流友好、速度快）
- REPL 交互模式降低重复输入成本
- 桌面 GUI 是可选增强（对视觉流友好、适合浏览分析）
- CLI 和 GUI 通过本地 HTTP API 通信，共享同一后端，数据一致

### 3.9 本地优先

- 所有数据存储在用户本地（不依赖云服务即可完整使用）
- 云端功能（Google Drive、Google Calendar）是可选的增强，而非必需的依赖
- API 服务器绑定 127.0.0.1（不暴露于网络）

### 3.10 中文原生

- 界面语言、提示信息全面中文化
- 中文路径、中文文件名、中文科目名均可正常处理
- 科目支持中文模糊匹配（"高数"→calculus、"大物"→physics）

## 四、用户典型使用场景

### 场景 A：日常学习记录

> 学完一章 → 打开终端 → `study log "高数: 完成多元函数微分"` → 关掉终端。或者直接在交互模式里敲。睡前 `study overview` 看一眼今日全局。

### 场景 B：考前突击

> `study countdown` 看还剩几天 → `study plan` 让 AI 生成复习日程 → `study pgen "高数" --sections all --total 20` 出题自测 → 对着 Dashboard 上的薄弱点列表针对性复习。

### 场景 C：新学期开始

> `study init` → 输入本学期课程 → 输入考试日期 → 完成。后续所有记录和复习都在已有框架下进行。

### 场景 D：回顾反思

> `study diary search "微分方程"` → 全文搜索过去所有日记中和微分方程有关的内容 → 发现三个月前就遇到过这个问题，对比当时的理解和现在的理解。

### 场景 E：碎片时间复习

> 手机打开 Google Calendar → 看到 AI 排好的今晚"概率论-贝叶斯公式（40分钟）"→ 从 Drive 下载对应的讲义 PDF → 开始复习。

## 五、不做的事情（明确边界）

- ❌ 不是在线课程平台 — 不托管教学视频、不在线做题
- ❌ 不是社交学习工具 — 没有好友、排行榜、打卡分享
- ❌ 不是团队协作工具 — 不提供多人共享、权限管理
- ❌ 不替代专业笔记软件 — 日记功能是简短的反思记录，不是 Obsidian/Notion 的替代品
- ❌ 不处理成绩/学分计算 — 聚焦"学习过程"的管理，不涉及"学习结果"的量化
- ❌ 不支持移动端 — 仅 Windows 桌面（CLI + GUI），不开发 iOS/Android 客户端
